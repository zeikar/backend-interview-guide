# Consistency Check — 구조 일관성 검증

면접 가이드 레포지토리의 README 목차, 파일 구조, 문서 간 참조의 일관성을 검증하는 스킬.

## 검증 항목

### 1. README ↔ 파일 매핑

프로젝트의 README 구조는 다음과 같다:

```
README.md (루트)
  ├── database/README.md
  ├── cloud/README.md
  ├── system-design/README.md
  └── programming/README.md
```

**검증 규칙:**

1. 루트 README.md의 목차에 있는 카테고리 디렉토리가 존재하는지 확인
2. 각 카테고리 README.md의 목차에 있는 `.md` 파일이 실제로 존재하는지 확인
3. 카테고리 디렉토리에 있는 `.md` 파일 중 README에 등록되지 않은 것이 있는지 확인

### 2. 문서 내 앵커 링크 검증

각 문서의 목차(## 목차)에 있는 앵커 링크가 문서 내 실제 헤딩과 일치하는지 확인한다.

**검증 방법:**
1. 목차의 `[텍스트](#앵커)` 패턴을 추출
2. 문서 내 `#`, `##`, `###` 헤딩을 추출
3. 앵커가 해당 헤딩으로 변환 가능한지 확인

### 3. 문서 간 상호 참조

문서 내에서 다른 문서를 참조하는 상대 링크가 유효한지 확인한다.

**검증 대상:**
- `[텍스트](파일명.md)` 형식의 같은 디렉토리 내 참조
- `[텍스트](../다른카테고리/파일명.md)` 형식의 카테고리 간 참조

### 4. 파일명 컨벤션

| 규칙 | 패턴 | 예시 |
|------|------|------|
| 소문자 | `[a-z-]+\.md` | `caching.md` |
| 하이픈 구분 | 복합어는 하이픈 | `cloud-computing.md` |
| 영문 | 한국어 파일명 금지 | `messaging-system.md` |
| README | 카테고리 인덱스 | `README.md` (대문자) |

## 워크플로우

### Step 1: 파일 목록 수집

```
1. Glob("**/*.md")로 전체 마크다운 파일 목록 수집
2. 각 카테고리 디렉토리의 README.md를 Read
3. README 목차에서 참조하는 파일 목록 추출
```

### Step 2: 매핑 검증

```
실제 파일 목록과 README 목차 참조를 비교:
- 목차에 있지만 파일 없음 → "누락 파일" 보고
- 파일 있지만 목차에 없음 → "미등록 파일" 보고
```

### Step 3: 링크 검증

```
각 .md 파일의 내부 링크를 추출하여:
- 앵커 링크 → 같은 문서 내 헤딩 존재 확인
- 파일 링크 → 대상 파일 존재 확인
```

### Step 4: 자동 수정

다음 문제는 자동으로 수정한다:
- README 목차의 오타성 링크 경로 (파일이 존재하는 경우)
- 새로 추가된 파일의 README 등록

다음 문제는 수정하지 않고 보고만 한다:
- 콘텐츠 파일 누락 (content-writer가 담당)
- 문서 내용의 기술적 오류 (content-reviewer가 담당)

### Step 5: 보고서 생성

검증 결과를 구조화된 보고서로 출력한다.

## 에러 핸들링

- 빈 디렉토리는 보고서에 명시한다
- 바이너리 파일은 건너뛴다
- README가 없는 카테고리 디렉토리는 경고로 보고한다
