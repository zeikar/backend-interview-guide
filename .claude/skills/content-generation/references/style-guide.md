# 콘텐츠 스타일 가이드

프로젝트의 기존 문서에서 추출한 구조적 패턴. 새 콘텐츠 작성 시 이 패턴을 따른다.

카테고리별 세부 스타일(포커스, 깊이, 코드 비중 등)은 이 문서에 정의하지 않는다. 작성 전에 해당 카테고리의 기존 문서 2~3개를 읽고 동적으로 파악하라.

## 제목 패턴

```markdown
# 한국어 주제명 (English Name)
```

h1은 문서당 1개. h2는 주요 섹션, h3는 세부 항목.

제품/기술명은 canonical form으로 유지한다 (AGENTS.md Writing Tips 참조):
- `# Redis` (O) — 제품명은 그대로
- `# 캐싱 (Caching)` (O) — 일반 개념은 병기
- `# 레디스 (Redis)` (X) — 과도한 번역 금지

---

## 목차 패턴

문서 최상단 (h1 아래)에 마크다운 링크 기반 목차를 배치한다.

```markdown
## 목차

- [섹션1](#한국어-앵커)
- [섹션2](#한국어-앵커)
  - [하위 섹션](#하위-앵커)
```

앵커는 한국어 그대로 사용한다 (GitHub 마크다운 호환).

## 설명 패턴

### 개념 정의

첫 문단에서 핵심 개념을 정의한다. 볼드로 주요 용어를 강조한다.

```markdown
**마이크로서비스 아키텍처(MSA)** 는 애플리케이션을 작고 독립적인 서비스 단위로
분리하여 개발, 배포, 운영할 수 있도록 설계하는 **분산 소프트웨어 아키텍처**입니다.
```

### 특징/장단점 리스트

볼드 라벨 + 콜론 + 설명 형식을 사용한다.

```markdown
- **설명:** 서버에서 자주 사용되는 데이터를 메모리나 디스크에 저장
- **장점:** 빠른 데이터 접근이 가능하며, 서버 측에서 일관성을 보다 쉽게 관리
- **단점:** 서버 리소스를 추가로 사용하며, 캐시의 크기가 제한될 수 있음
```

### 번호 리스트 + 볼드 제목

순서가 있는 항목은 다음 형식을 따른다.

```markdown
1. **간결하고 읽기 쉬운 문법**
   - 복잡한 문법 없이, 직관적인 코드 작성 가능.

2. **강력한 동시성 지원**
   - **Goroutines**와 **Channels**를 통해 경량 동시성 처리.
```

### 구분선

주요 섹션(h2) 사이에 `---`를 넣어 시각적으로 구분한다. h3 사이에는 넣지 않는다.

## 비교표 패턴

```markdown
| **특징** | **A** | **B** |
| --- | --- | --- |
| 구조 | A의 설명 | B의 설명 |
| 배포 | A의 설명 | B의 설명 |
```

- 첫 번째 열은 비교 기준
- 헤더와 내용은 모두 한국어
- 셀 내용은 간결하게 (1~2줄)
- 유사 개념이 있으면 반드시 비교표를 포함한다

## 코드 예시 패턴

언어 태그를 명시하고, 핵심 라인에 주석을 단다.

```markdown
```go
// Goroutine 생성 예시
go func() {
    fmt.Println("Hello from Goroutine")
}()
`` `
```

- 언어 태그 필수 (```go, ```javascript, ```python 등)
- 주석으로 핵심 포인트 설명
- 안티패턴이 아닌 실행 가능한 코드
