## 제목 패턴

```markdown
# 한국어 주제명 (English Name)
```

h1은 문서당 1개. h2는 주요 섹션, h3는 세부 항목.

**예시:**
```markdown
# 캐싱 (Caching)
## 캐시의 유형
### 클라이언트 측 캐시(Client-side Cache)
```

---

## 목차 패턴

문서 최상단 (h1 아래)에 마크다운 링크 기반 목차를 배치한다.

```markdown
## 목차

- [섹션1](#한국어-앵커)
- [섹션2](#한국어-앵커)
  - [하위 섹션](#하위-앵커)
```

앵커는 한국어 그대로 사용한다 (GitHub 마크다운 호환).

## 설명 패턴

### 개념 정의

첫 문단에서 핵심 개념을 정의한다. 볼드로 주요 용어를 강조한다.

```markdown
**마이크로서비스 아키텍처(MSA)** 는 애플리케이션을 작고 독립적인 서비스 단위로 
분리하여 개발, 배포, 운영할 수 있도록 설계하는 **분산 소프트웨어 아키텍처**입니다.
```

### 특징/장단점 리스트

볼드 라벨 + 콜론 + 설명 형식을 사용한다.

```markdown
- **설명:** 서버에서 자주 사용되는 데이터를 메모리나 디스크에 저장
- **장점:** 빠른 데이터 접근이 가능하며, 서버 측에서 일관성을 보다 쉽게 관리
- **단점:** 서버 리소스를 추가로 사용하며, 캐시의 크기가 제한될 수 있음
```

### 번호 리스트 + 볼드 제목

순서가 있는 항목은 다음 형식을 따른다.

```markdown
1. **간결하고 읽기 쉬운 문법**
   - 복잡한 문법 없이, 직관적인 코드 작성 가능.

2. **강력한 동시성 지원**
   - **Goroutines**와 **Channels**를 통해 경량 동시성 처리.
```

## 비교표 패턴

```markdown
| **특징** | **A** | **B** |
| --- | --- | --- |
| 구조 | A의 설명 | B의 설명 |
| 배포 | A의 설명 | B의 설명 |
```

- 첫 번째 열은 비교 기준
- 헤더와 내용은 모두 한국어
- 셀 내용은 간결하게 (1~2줄)

## 카테고리별 예시

### database 카테고리

핵심: 대규모 트래픽 관점의 데이터베이스 기술. 성능 수치, 트레이드오프 분석 포함.

```markdown
# 캐싱 (Caching)

## 목차
- [캐싱](#캐싱)
- [캐시의 유형](#캐시의-유형)
...

# 캐싱

캐시는 데이터를 더 빠르게 접근할 수 있는 위치에 저장하여...

## 클라이언트 측 캐시(Client-side Cache)
- **설명:** ...
- **장점:** ...
- **단점:** ...
```

### cloud 카테고리

핵심: 클라우드 인프라와 분산 시스템. 아키텍처 패턴, 운영 관점 포함.

### system-design 카테고리

핵심: 시스템 설계 면접 준비. 접근 방법론, 트레이드오프 분석, 실제 면접 질문 시나리오 포함.

### programming 카테고리

핵심: 언어별 특성과 동시성. 코드 예시 비중이 높다. 자료구조/알고리즘은 개념 + 복잡도 분석.
